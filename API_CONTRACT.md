# API Interface Contract

This document defines the public HTTP contract for client applications that integrate with the Client License Validation API.

The contract is designed to be technology-neutral. Any client application that can send HTTPS JSON requests can integrate with it.

## Base URL

Local development:

```text
http://localhost:5212
```

Azure App Service:

```text
https://<your-app-service-name>.azurewebsites.net
```

Production clients should use HTTPS.

## Compatibility Rules

The API follows these compatibility rules:

- Existing request fields will not be removed without a new API version.
- Existing response fields will not be removed without a new API version.
- New optional request fields may be added.
- New response fields may be added.
- Clients must ignore unknown response fields.
- Server must ignore unknown request fields.
- Clients should use `code` for programmatic handling and `message` for user-facing display.
- `extra` is reserved for forward-compatible client metadata.

## POST /api/v1/license/validate

Validates whether a deployed client application is licensed to continue login/access.

The API uses server UTC time. Client machine time is not trusted for expiry decisions.

### Request Headers

```http
Content-Type: application/json
Accept: application/json
```

### Request Object

```json
{
  "schemaVersion": "1.0",
  "clientId": "pharmacloud-fakhirgroup-com",
  "applicationId": "pharmacloud",
  "environment": "production",
  "activationId": "pharmacloud-fakhirgroup-com-pharmacloud-production",
  "installationId": "2f4b5f84-91d6-4b28-b0a8-59bdf57c927a",
  "appVersion": "1.0.0",
  "requestId": "7eb18b12-7f3d-42b5-bc39-29f993a8b777",
  "extra": {
    "tenantCount": 25,
    "region": "UAE"
  }
}
```

### Request Fields

| Field | Type | Required | Description |
| --- | --- | --- | --- |
| `schemaVersion` | string | recommended | Client request schema version. Current value: `1.0`. |
| `clientId` | string | yes | Customer, reseller, or company identifier. Same across production/test environments for the same client. |
| `applicationId` | string | yes | Product/application identifier, for example `pharmacloud`. |
| `environment` | string | yes | Deployment environment, for example `production`, `test`, `staging`, or `demo`. |
| `activationId` | string | yes | Unique activation record for client + application + environment. |
| `installationId` | string | yes | Fixed GUID for this deployed application instance. |
| `appVersion` | string | no | Client application version/build. Useful for diagnostics. |
| `requestId` | string | no | Unique request correlation id generated by client. |
| `extra` | object | no | Future-compatible metadata object. Server may ignore fields it does not need. |

### Response Object

```json
{
  "valid": true,
  "status": "active",
  "code": "LICENSE_ACTIVE",
  "message": "License active.",
  "serverTimeUtc": "2026-06-12T16:00:00.0000000+00:00",
  "tokenValidUntilUtc": "2026-06-13T16:00:00.0000000+00:00",
  "features": [
    "core",
    "reports"
  ],
  "limits": {
    "maxTenants": null,
    "maxUsers": null
  },
  "responseTimeMs": 0.74,
  "token": "eyJhbGciOiJFUzI1NiIsInR5cCI6IkpXVCIsImtpZCI6ImxpY2Vuc2luZy1rZXktMDAxIn0..."
}
```

### Response Fields

| Field | Type | Description |
| --- | --- | --- |
| `valid` | boolean | `true` when client should proceed; `false` when client should block login/access. |
| `status` | string | Business status: `active`, `expired`, `suspended`, or `invalid`. |
| `code` | string | Stable programmatic result code. |
| `message` | string | User-facing message controlled centrally by the API. |
| `serverTimeUtc` | datetime | API server UTC time. Client should store this as last known server time. |
| `tokenValidUntilUtc` | datetime | Signed token validity end time. Current policy: one day from validation. |
| `features` | string array | Enabled modules/features. Placeholder for future client enforcement. |
| `limits` | object | License limits such as tenant/user limits. Placeholder for future client enforcement. |
| `responseTimeMs` | number | Approximate server-side response generation time in milliseconds. |
| `token` | string | ES256 signed JWT-style validation token. |

### Business Status Values

| Status | Meaning | Client Behavior |
| --- | --- | --- |
| `active` | License is valid. | Allow login/access. |
| `expired` | License expiry date has passed according to server UTC time. | Block and show `message`. |
| `suspended` | License is centrally disabled. | Block and show `message`. |
| `invalid` | Request/license is unknown, malformed, or not allowed. | Block and show `message`. |

### Result Codes

| Code | Valid | Meaning |
| --- | --- | --- |
| `LICENSE_ACTIVE` | true | License is active. |
| `LICENSE_EXPIRED` | false | License has expired. |
| `LICENSE_SUSPENDED` | false | License is suspended. |
| `LICENSE_NOT_FOUND` | false | No matching license was found for `clientId` + `applicationId`. |
| `LICENSE_ENVIRONMENT_NOT_ALLOWED` | false | Environment is not allowed for this license. |
| `LICENSE_ACTIVATION_NOT_ALLOWED` | false | Activation id is not allowed for this license. |
| `LICENSE_INSTALLATION_NOT_ALLOWED` | false | Installation id is not allowed for this license. |
| `LICENSE_INVALID_STATUS` | false | Configured license status is not active/expired/suspended. |
| `LICENSE_REQUEST_INVALID` | false | Required request fields are missing or invalid. |

### HTTP Status Codes

The validation endpoint returns `200 OK` for known business outcomes, including denied licenses.

```text
200 OK           Request processed. Check `valid`, `status`, and `code`.
400 Bad Request  Malformed JSON or invalid HTTP request shape.
500 Server Error Unexpected API failure.
```

Clients should not treat HTTP `200` as license success. They must check:

```text
valid == true
```

## Token Contract

`token` is a compact ES256 JWT-style signed token.

Header example:

```json
{
  "alg": "ES256",
  "typ": "JWT",
  "kid": "licensing-key-001"
}
```

Payload contains the license validation result:

```json
{
  "schemaVersion": "1.0",
  "valid": true,
  "status": "active",
  "code": "LICENSE_ACTIVE",
  "message": "License active.",
  "clientId": "pharmacloud-fakhirgroup-com",
  "applicationId": "pharmacloud",
  "environment": "production",
  "activationId": "pharmacloud-fakhirgroup-com-pharmacloud-production",
  "installationId": "2f4b5f84-91d6-4b28-b0a8-59bdf57c927a",
  "serverTimeUtc": "2026-06-12T16:00:00.0000000+00:00",
  "tokenValidUntilUtc": "2026-06-13T16:00:00.0000000+00:00",
  "features": ["core", "reports"],
  "limits": {
    "maxTenants": null,
    "maxUsers": null
  }
}
```

Client applications may verify the token using the public key corresponding to `kid`.

## Recommended Client Decision Logic

```text
If HTTP request fails:
  block login/access
  show local fallback activation failed message

If HTTP returns 200:
  if response.valid == true:
    cache signed token until tokenValidUntilUtc
    allow login/access
  else:
    show response.message
    block login/access

If local machine time is earlier than last stored serverTimeUtc:
  ignore cached token
  call API again
```

## Versioning

Current version:

```text
/api/v1/license/validate
```

Breaking changes should be introduced under a new route:

```text
/api/v2/license/validate
```

Non-breaking changes may be added to v1 as optional fields.
